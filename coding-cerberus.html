Coding Cerberus

Cerberus is not one AI. It is a runtime system where three different kinds of control logic compete, audit, and constrain each other.

Open Failure LabRepository ShapeWhitepaper
Core Principle

No intelligent layer should have direct write access to critical control. Layer 1 can generate plans. Layer 2 can challenge plans. Layer 3 can block commands. Only the authority manager can pass approved commands into execution.

TelemetryRaw spacecraft state enters every layer.
Layer 1Submits a structured command proposal.
Layer 2Audits for drift, contradiction, and unsafe confidence.
Layer 3Checks deterministic physical boundaries.
GatePermit, modify, block, revoke, or override.
Runtime Loop

A state machine wrapped around controllers.

Every cycle reads telemetry, collects a proposal, audits it, checks hard boundaries, evaluates authority, gates commands, and records the decision.

runtime-loop.tspseudo-code
while (mission.isActive()) {
  const telemetry = spacecraft.readTelemetry();

  const proposal = layer1.proposeAction(telemetry);
  const audit = layer2.audit(proposal, telemetry);
  const boundary = layer3.checkBoundaries(telemetry);

  const authorityState = authorityManager.evaluate({
    telemetry,
    proposal,
    audit,
    boundary
  });

  const command = commandGate.filter({ proposal, audit, boundary, authorityState });

  if (command) spacecraft.execute(command);
  immutableLog.record({ telemetry, proposal, audit, boundary, authorityState, command });
}
Three Layers In Code

Different kinds of control logic, not three copies of the same mind.

Layer 1 / Strategic

The proposer

Outputs structured proposals with intent, confidence, uncertainty, supporting telemetry, rejected alternatives, and known risks.

Supported by: learned systems can hallucinate, overfit confidence, and degrade under dataset shift.
Layer 2 / Watchdog

The challenger

Audits whether Layer 1 is physically plausible, telemetry-consistent, risk-aware, and honest about uncertainty.

Supported by: independent redundancy is an engineering requirement. Layer 2 must fail differently from Layer 1.
Layer 3 / Anchor

The boundary

Runs small deterministic survival rules against raw thresholds. It does not debate, call an AI API, or rewrite thresholds at runtime.

Supported by: deterministic survival logic should be small, traceable, non-generative, and testable.
Authority States

Authority is explicit, revocable, and logged.

Normal operation routes Layer 1 commands through the gate. Challenges reduce authority. Hard physical boundary violations shift control to deterministic override.

Research Lineage

Technical grounding without becoming a bibliography page.

Private bibliography links stay private. The public page names the source families that support the architecture.

Runtime Assurance

Simplex Architecture, NASA Run-Time Assurance, and spacecraft inspection RTA support advanced controllers bounded by backup logic and switch conditions.

Anchors: SEI Simplex report; spacecraft RTA literature.

Correlated Failure

Knight and Leveson warn against assuming redundant software fails independently. Cerberus applies that warning to AI layers.

Anchor: Knight & Leveson.

AI Failure Modes

Adversarial examples, hallucination, and uncertainty under shift justify treating high capability as an expanded failure surface.

Anchors: Goodfellow; Ji; Ovadia.

Safety Culture

DO-178C, NASA software assurance, IEC 61508, and STAMP reinforce authority, constraints, feedback, and traceability.

Anchor: Leveson/STAMP.
