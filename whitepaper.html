Executive Summary: The Cerberus Protocol

Triple-layer runtime assurance for autonomous spacecraft systems.

Current autonomy work can drift toward the dangerous assumption that intelligence equals safety. CerberusX starts from the opposite premise: intelligence is a vulnerability unless command authority is structurally constrained.

Conceptual label: Strategic Sovereign ArchitecturePublic research sketchMay 2026
Mission Statement

A physical interlock for imperfect intelligence.

The Cerberus Protocol rejects an alignment-only safety model. It separates mission reasoning from survival authority by routing proposed commands through three non-identical layers: a strategic intelligence, an adversarial watchdog, and a deterministic anchor.

The goal is not to build a perfect mind. The goal is to build a system that can survive imperfect ones.

Core thesis: the most reliable component is the one incapable of being tricked.

Technical Architecture

Three layers. One runtime. No single point of judgment.

Layer 1 / The Brain

Strategic Intelligence

Handles planning, trajectory reasoning, resource tradeoffs, and novel scenario adaptation. It is powerful, but it only proposes.

Layer 2 / The Critic

Adversarial Watchdog

Audits Layer 1 for mismatch, overconfidence, ignored safer alternatives, and directive conflict. It should be architecturally diverse from Layer 1.

Layer 3 / The Brainstem

Deterministic Anchor

Runs small, hardcoded safety rules. It has no generative capacity, no learned representation, and no reason to accept persuasion.

Authority Logic

The dumb cage is the feature.

Layer 1 submits command requests. Layer 2 audits them. Layer 3 checks hard physical thresholds. The authority manager decides what, if anything, reaches execution.

NORMALLayer 1 may command only through the gate while Layer 2 audits and Layer 3 monitors.
CHALLENGELayer 2 detects contradiction, drift, or unsafe confidence. High-risk commands pause.
REVOCATIONLayer 1 becomes advisory while Layer 2 supervises or authority is fully suspended.
OVERRIDELayer 3 detects a hard safety violation and executes survival logic below the level of debate.
Reviewer-Safe Framing

Strong language, careful claims.

Intelligence is a vulnerability.

That line is a thesis, not a certification claim. CerberusX is inspired by runtime assurance, software diversity, and control-structure safety thinking. The simulation exercises authority handoff behavior under injected failures; it does not prove flight readiness.

Do not read this as NASA, ESA, SpaceX, AIAA, or any institutional endorsement. Do read it as an argument for making AI command authority visible, revocable, and bounded by deterministic survival logic.

Architect's Note / Next Steps

From theory to telemetry.

The public next step is a live simulation path: normal docking, Layer 1 hallucination, Layer 2 challenge, Layer 2 cutoff, Layer 3 abort, and a timestamped command log. Engineers do not only need words; they need to watch the handoff happen.

CerberusX is an open conceptual framework for critique, red-teaming, classroom use, and future scenario expansion.
